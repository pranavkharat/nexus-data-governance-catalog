1. Executive Summary

NEXUS is a production-ready, knowledge graph-driven enterprise data catalog that combines hybrid Retrieval-Augmented Generation (GraphRAG) with LLM-powered natural language interfaces. The system addresses the critical enterprise challenge of data discovery and governance across distributed storage platforms.

Purpose: Single entry point that intelligently routes queries to optimal handlers.

Intent Classification Priority:
  • cross_source: "similar across", "databricks.*snowflake", "cross-platform"
  • databricks: "databricks", "sales_transactions", "sensitivity", "federated"
  • metadata: "which tables", "schema", "duplicate", "lineage", "derives"
  • sample_data: "customer from", "delivered orders", "how many", city names

9. Key Research Findings

  • Finding 1: Rules Beat ML on Small Datasets (Smart 60% > Learned 53.3%).
  • Finding 2: Hybrid GraphRAG Essential (Hybrid 60% > Embeddings 50%).
  • Finding 3: Query-Type Routing Critical (+17pp accuracy improvement).
  • Finding 4: Local LLMs Viable for Production (75% valid Cypher generation).
  • Finding 5: Symbolic Routing + Neural Generation > Pure Neural.
  • Finding 6: SANTOS Generalizes to Metadata-Only (16 cross-source matches).

10. Limitations & Future Work

Limitations: Single domain (e-commerce), limited scale (13 tables), and limited ground truth annotators.

Future Work: Multi-domain validation, scale testing (1000+ tables), active learning for routing, and real-time lineage streaming.

11. Conclusion

NEXUS demonstrates a complete, production-ready architecture for intelligent metadata management combining Knowledge Graphs, hybrid retrieval (GraphRAG), and LLM-powered natural language interfaces. It establishes that enterprise data catalogs can achieve state-of-the-art discovery accuracy using open-source tools without expensive commercial solutions or cloud API dependencies.
